# RSS → Discord 通知

好きな個人サイトのRSS/Atomフィードを定期的にチェックして、新着記事があったらDiscordのチャンネルに通知するだけの、シンプルなスクリプトです。GitHub Actionsの上で動くので、自分のPCやサーバーを常時起動しておく必要はありません。

## 仕組み

4時間おきにGitHub Actionsが起動し、`feeds.txt` に書かれたフィードを全部チェックします。前回までに見た記事は `seen.json` に記録されていて、そこに無い記事だけがDiscordに通知されます。

## セットアップ

### 1. Discordのウェブフックを用意する

通知を送りたいチャンネルの設定から「連携サービス」→「ウェブフック」→「新しいウェブフック」でURLを発行し、コピーしておきます。

### 2. このフォルダをGitHubリポジトリにする

GitHubで新しい空のリポジトリを作り、このフォルダの中身一式(`.github` フォルダも含めて全部)をアップロードします。リポジトリのページにある「Add file」→「Upload files」から、フォルダごとドラッグ&ドロップすれば、構造を保ったままアップロードできます(gitコマンドを使える人は、普通に `git init` してpushしてもOKです)。

公開・非公開どちらのリポジトリでも動きますが、ひとつだけ注意点があります。

- 公開リポジトリ → GitHub Actionsの実行時間が完全無料・無制限。ただし `feeds.txt`(どのサイトを見ているか)が誰でも見える状態になります。
- 非公開リポジトリ → 無料枠は月2,000分まで。4時間おきの実行ならこの用途では全然余裕で収まりますが、`feeds.txt` の中身は自分だけに見える状態にできます。

ウェブフックのURL自体は次のステップで「シークレット」として登録するので、どちらのリポジトリを選んでも外部に漏れることはありません。

### 3. ウェブフックURLをシークレットに登録する

リポジトリの `Settings` → `Secrets and variables` → `Actions` → `New repository secret` を開き、

- Name: `DISCORD_WEBHOOK_URL`
- Value: 手順1でコピーしたウェブフックURL

として保存します。

### 4. 見たいサイトを feeds.txt に追加する

`feeds.txt` を開いて、監視したいサイトのRSS/AtomフィードのURLを1行ずつ追加してください。サイトのフィードURLが分からない場合は、`サイトのURL/feed` や `サイトのURL/rss.xml` を試すか、ページのソースコードの中から `application/rss+xml` を検索してみてください。

### 5. 動作確認

リポジトリの `Actions` タブを開き、`Check RSS feeds` ワークフローを選んで `Run workflow` から手動実行できます。実行ログで各フィードがちゃんと読み込めているか確認してください。

**初回実行では、その時点にある記事は「既読」として記録されるだけで、Discordへの通知は来ません。** 起動した瞬間に過去の記事が大量に通知されるのを防ぐためです。次回の実行(または該当サイトに新しい記事が出たあとの実行)から、新着だけが通知されます。

## 新しいサイトを追加したいとき

`feeds.txt` にURLを1行足してコミットするだけです。コードを触る必要はありません。

## 通知の間隔を変えたいとき

`.github/workflows/check-feeds.yml` の中の `cron: '0 */4 * * *'` を編集してください。GitHub Actionsのスケジュールは協定世界時(UTC)基準なので、日本時間(JST)で考えたいときは9時間分ずらして計算する必要があります。

## ローカルで試したい場合

```
pip install feedparser requests
export DISCORD_WEBHOOK_URL="あなたのウェブフックURL"
python check_feeds.py
```
